# Feature Requirements Document (FRD 7): Hybrid Care Bridge (Labs, Pharmacies, Hospitals)

**Product:** HealthNet Operating System (HOS)  
**FRD Version:** 1.0  
**Date:** January 2026  

## Objective
- Seamlessly migrate secure patient data from the virtual platform to hospital/clinic EMR platform facilitating follow-up management (labs, imaging, pharmacies) via orders, referrals and status tracking.

## Users
- Referrers: Professionals  
- Fulfillers: Verified labs, pharmacies, hospitals  
- Patients: Track status

## Functional Requirements
- For hospitals with branded virtual platforms, easy migration of patient data (both biodata and clinical information) into hospital/clinic EMR.
- Order creation at choice physical center from the virtual platform: Lab tests, imaging, prescriptions fulfillment, hospital referrals.
- Workflow: Send order → fulfiller accepts → status updates (pending, in-progress, completed, results ready).
- Integration plugin: Institutions enable/disable services, provide API endpoints (details per CTO specification).
- Patient portal: View orders, results (secure upload/view), pharmacy pickup/delivery status.

## Non-Functional Requirements
- Interoperability: FHIR-based where possible.

## Regulatory & Compliance
- MDCN/PCN: Prescription integrity, referral tracking.
- NDPC: Secure result sharing with consent.

## Constraints & Safety
- Manual confirmation step for high-risk orders.
- Fallback: Fax/email for non-integrated partners.

## Success Criteria
- ≥85% order completion within 48 hours.
- Zero untracked critical results.
